DriveMatrix API v1 – Documentación para Developers

Creado@2 de diciembre de 2025 14:14

Base URL: /api

Formato de datos: JSON para request y response

Content-Type: application/json

Backend: Flask + MongoDB

Autenticación: JWT (Bearer Token) en endpoints protegidos

Password: Hasheadas con bcrypt

Fechas: ISO 8601 (YYYY-MM-DDTHH:MM:SS)

1. Endpoints Generales

1.1 Documentación

GET /api

Devuelve la documentación HTML de la API.

1.2 Catch-all

GET /<any_path>

Redirige a /api.

2. Endpoints de Usuarios

2.1 Crear Usuario

POST /api/user/create

Crea un usuario en MongoDB.

Request JSON:

{
  "nombre": "Lucas",
  "email": "lucas@example.com",
  "password": "MiPassword123"
}

Response 201:

{
  "message": "Usuario creado",
  "user_id": "6510c5d5f5a3e3e5d2b9e1c3"
}

Errores comunes:

  • 400 Bad Request si falta algún campo:
{"error": "nombre, email y contraseña son requeridos"}

2.2 Login de Usuario

POST /api/user/login

Genera JWT para el usuario.

Request JSON:

{
  "email": "lucas@example.com",
  "password": "MiPassword123"
}

Response 200:

{
  "token": "<JWT_TOKEN>"
}

Errores comunes:

  • 400: falta algún campo
  • 403: contraseña incorrecta
  • 404: usuario no encontrado

2.3 Consultar Usuario

POST /api/user/show

Busca un usuario por email o ID.

Request JSON:

{
  "emailOrId": "lucas@example.com"
}

Response 200:

{
  "_id": "6510c5d5f5a3e3e5d2b9e1c3",
  "nombre": "Lucas",
  "email": "lucas@example.com",
  "purchases_history": [],
  "wishlist": [],
  "created_at": "2025-12-02T11:00:00"
}

Errores comunes:

  • 400: falta emailOrId
  • 404: usuario no encontrado

2.4 Actualizar Usuario

PATCH /api/user/update/<user_id>

Actualiza nombre, password o wishlist. Solo campos válidos serán actualizados.

Request JSON ejemplo:

{
  "nombre": "Lucas Fernández",
  "password": "NuevaPassword456",
  "wishlist_items": [{"vehicle_vin": "VIN12345"}]
}

Response 200:

{"message": "Usuario actualizado correctamente"}

Errores comunes:

  • 400: no hay campos válidos
  • 404: usuario no encontrado

2.5 Eliminar Usuario

DELETE /api/user/delete

Requiere JWT.

Headers:

Authorization: Bearer <JWT_TOKEN>

Response 200:

{"message": "Usuario eliminado correctamente"}

Errores comunes:

  • 401: token requerido, inválido o expirado
  • 500: no se pudo eliminar

3. Endpoints Wishlist

3.1 Agregar a Wishlist

POST /api/user/wishlist/add

Requiere JWT.

Request JSON:

{
  "vehicle_vin": "VIN12345"
}

Response 200:

{
  "message": "Vehículo agregado a wishlist",
  "item": {"vehicle_vin": "VIN12345", "added_at": "2025-12-03T00:00:00"}
}

Errores:

  • 400: falta vehicle_vin o vehículo ya agregado

3.2 Eliminar de Wishlist

POST /api/user/wishlist/remove

Requiere JWT.

Request JSON:

{
  "vehicle_vin": "VIN12345"
}

Response 200:

{"message": "Vehículo eliminado de wishlist"}

Errores:

  • 400: falta vehicle_vin
  • 404: vehículo no encontrado en wishlist

3.3 Listar Wishlist

GET /api/user/wishlist

Requiere JWT.

Response 200:

{"wishlist": [{"vehicle_vin": "VIN12345", "added_at": "2025-12-03T00:00:00"}]}

4. Endpoints Vehículos

4.1 Listar todos los vehículos

GET /api/auto/listings

Response 200:

[
  {
    "_id": "64f1a5...",
    "vehiculos": [
      {"vin": "VIN123", "make": "Toyota", "model": "Corolla", ...}
    ]
  }
]

4.2 Filtrar vehículos

GET /api/auto/listings/filter

Parámetros opcionales (query string):

doors, drivetrain, engine, fuel, make, model, seats, transmission

Ejemplo:

/api/auto/listings/filter?make=Toyota&doors=4

Response 200:

[
  {"vin": "VIN123", "make": "Toyota", "model": "Corolla", "doors": 4, ...}
]

4.3 Actualizar Mongo con Auto.dev

GET /auto/actualizarMongo

Obtiene vehículos desde auto.dev y los guarda en MongoDB.

4.4 Obtener imágenes por VIN

GET /auto/image/<vin>

Response 200:

[
  "https://link_a_imagen.jpg",
  ...
]

5. Endpoints Compras

5.1 Crear compra

POST /api/purchase/create

Requiere JWT.

Request JSON:

{"vehicle_vin": "VIN12345"}

Response 200:

{
  "Venta Creada": {
    "_id": "64f1a5...",
    "ref_user_id": "6510c5d5f5a3e3e5d2b9e1c3",
    "ref_vehicle_vin": "VIN12345",
    "date": "2025-12-03T00:00:00"
  }
}

Errores comunes:

  • 400: falta vehicle_vin
  • 404: vehículo no encontrado

6. Seguridad y Autenticación

  • JWT: Se envía en Authorization: Bearer <token>
  • Expiración: 2 horas
  • Password: Hasheadas con bcrypt
  • Mongo ObjectId: Devuelto como string en JSON
