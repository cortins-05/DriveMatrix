DriveMatrix API v1 – Documentación para Developers

Creado@2 de diciembre de 2025 14:14

Formato de datos:

Todos los endpoints usan JSON para request y response.

Content-Type: application/json

1. Descripción General

DriveMatrix API permite gestionar usuarios de la aplicación, incluyendo creación, consulta, actualización y eliminación. La API usa MongoDB como backend y Flask para el servidor.

  • Autenticación: Ninguna por ahora (future: JWT).
  • Passwords: Almacenadas con hash seguro (bcrypt).
  • Tipo de API: RESTful, basada en recursos /user.

2. Endpoints de Usuarios

2.1 Crear Usuario

POST /user/create

Crea un nuevo usuario en la base de datos.

Parámetros JSON

CampoTipoObligatorioDescripción
nombrestringSíNombre completo del usuario
emailstringSíEmail único del usuario
passwordstringSíContraseña (será hasheada internamente)

Ejemplo de Request

{
  "nombre": "Lucas",
  "email": "lucas@example.com",
  "password": "MiPassword123"
}

Ejemplo de Response

201 Created

{
  "message": "Usuario creado",
  "user_id": "6510c5d5f5a3e3e5d2b9e1c3"
}

Errores comunes

  • 400 Bad Request: Si faltan campos obligatorios.
{
  "error": "nombre, email y contraseña son requeridos"
}

2.2 Consultar Usuario

POST /user/show

Busca un usuario por email o Mongo ObjectId.

Parámetros JSON

CampoTipoObligatorioDescripción
emailOrIdstringSíEmail o _id del usuario

Ejemplo de Request

{
  "emailOrId": "lucas@example.com"
}

Ejemplo de Response

200 OK

{
  "_id": "6510c5d5f5a3e3e5d2b9e1c3",
  "nombre": "Lucas",
  "email": "lucas@example.com",
  "purchases_history": [],
  "wishlist": [],
  "created_at": "2025-12-02T11:00:00"
}

Errores comunes

  • 404 Not Found: Usuario no existe.
{
  "error": "Usuario no encontrado"
}

2.3 Actualizar Usuario

POST /user/update

Actualiza información de un usuario. El password actual debe ser enviado para validar la operación.

Parámetros JSON

CampoTipoObligatorioDescripción
emailstringSíEmail del usuario a actualizar
passwordstringSíContraseña actual para verificación
updateobjectSíCampos a actualizar (nombre, password)

Ejemplo de Request

{
  "email": "lucas@example.com",
  "password": "MiPassword123",
  "update": {
    "nombre": "Lucas Fernández",
    "password": "NuevaPassword456"
  }
}

Ejemplo de Response

200 OK

{
  "message": "Usuario actualizado"
}

Errores comunes

  • 401 Unauthorized: Password incorrecta.
  • 404 Not Found: Usuario no existe.

2.4 Eliminar Usuario

POST /user/delete

Elimina un usuario de la base de datos. Se requiere la contraseña para verificar identidad.

Parámetros JSON

CampoTipoObligatorioDescripción
emailstringSíEmail del usuario
passwordstringSíContraseña actual

Ejemplo de Request

{
  "email": "lucas@example.com",
  "password": "MiPassword123"
}

Ejemplo de Response

200 OK

{
  "message": "Usuario eliminado"
}

Errores comunes

  • 401 Unauthorized: Password incorrecta.
  • 404 Not Found: Usuario no existe.

3. Notas Técnicas

  1. Contraseñas: Se guardan como hash bytes (bcrypt). Para validar, siempre usar:
bcrypt.checkpw(password.encode("utf-8"), user["password"])
  1. IDs de Mongo: _id se devuelve como string en JSON.
  1. Content-Type: Todos los requests deben enviar application/json.
  1. Fechas: Se devuelven en formato ISO 8601 (YYYY-MM-DDTHH:MM:SS).
  1. Futuro: Se puede agregar JWT para autenticación, endpoints de wishlist y compras, y filtros de búsqueda avanzados.
